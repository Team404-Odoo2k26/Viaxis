2. Database Configuration
------------------------------------------------------------------------
Next.js connects directly to PostgreSQL via connection settings defined in:
  WEBSITE/frontend/.env.local

Connection String (Default):
  DATABASE_URL=postgresql://postgres@localhost:5432/transitops

Prisma Schema (Prisma client generated inside node_modules):
  WEBSITE/backend/prisma/schema.prisma

------------------------------------------------------------------------
3. Database Setup Commands
------------------------------------------------------------------------
To wipe the database, rebuild the tables schema, and load seed data:
  * From WEBSITE/frontend:
    npm run db:setup

  * Or execute manually:
    psql -U postgres -f WEBSITE/backend/database/schema.sql
    psql -U postgres -d transitops -f WEBSITE/backend/database/seed.sql

------------------------------------------------------------------------
4. Local Dev Servers
------------------------------------------------------------------------
To launch the unified Next.js App Server (Frontend UI + Database API layer):
  - On Windows: run `start.bat`
  - On macOS/Linux: run `./start.sh`

The Next.js server runs on:
  http://localhost:3000
